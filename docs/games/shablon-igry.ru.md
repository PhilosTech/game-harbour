# Как устроен шаблон игры (для ведущего)

Простое объяснение: что такое сцены, персонажи, характеристики и как проходит партия.
Техническая спека на английском: [game-creation.md](./game-creation.md).

Пример готовой игры: [Эльдорадо](./el-dorado.md).

---

## Это не D&D

Game Harbour **не копирует правила D&D**. Мы берём от настольных игр только **поток**:

1. Ведущий готовит сценарий (шаблон).
2. Игроки заходят в комнату.
3. Каждый получает роль / персонажа.
4. Ведущий ведёт по **сценам** - читает, задаёт ситуации, иногда бросают кубик.
5. История идёт вперёд, финал решает стол (ведущий + игроки).

Нет обязательных рас, классов, заклинаний, сложных таблиц. Всё проще и ближе к **сюжетной игре с ведущим**, а не к учебнику на 300 страниц.

---

## Два уровня: шаблон и сессия

|                 | Шаблон игры                               | Живая сессия                              |
| --------------- | ----------------------------------------- | ----------------------------------------- |
| **Что это**     | Заготовка, которую ведущий пишет один раз | Один конкретный вечер с реальными людьми  |
| **Пример**      | «В поисках Эльдорадо» с 10 сценами        | Комната `ABC123`, 4 игрока, суббота 20:00 |
| **Сколько раз** | Одна игра - много сессий                  | Каждый «Начать сессию» = новая комната    |

Один шаблон могут одновременно вести разные ведущие в **разных комнатах**. Создание игры не «занимает» её для других.

## Из чего состоит шаблон

### Уже есть в приложении

| Блок                     | Зачем                                                         |
| ------------------------ | ------------------------------------------------------------- |
| **Название и описание**  | Карточка в дашборде; игроки видят контекст                    |
| **Сцены**                | Этапы сюжета: текст, фон (URL), задания, заметки ведущего     |
| **Слоты героев**         | Список ролей; у каждого слота своя сильная сторона и слабость |
| **Общие характеристики** | 3+ параметра; игрок в лобби раскидывает случайный пул очков   |
| **Карточка персонажа**   | Попап у игрока: роль, сила/слабость, характеристики           |
| **Приват / публичный**   | Только вы или все ведущие                                     |

### Планируем добавить

| Блок                      | Зачем                               |
| ------------------------- | ----------------------------------- |
| **Загрузка обложки игры** | Позже (сейчас только картинки сцен) |
| **Аватар героя**          | Картинка на карточке персонажа      |

Всё это **привязано к шаблону игры**. В другой игре - другой список героев и другие характеристики.

## Сцены - что это на самом деле

**Сцена** - один **этап сюжета**, один «ход картины»:

- вступление в офисе спонсора;
- лодка по реке;
- встреча с конкурентами;
- находка в руинах.

Ограничения по количеству **нет** - сколько придумает ведущий. Порядок задаётся номером (1, 2, 3…).

Ведущий **сам** решает, когда перейти к следующей сцене. Приложение не проверяет «выполнили задание» - это живая игра за столом (только в телефонах).

### Зачем «ключ сцены»?

**Ключ** (`briefing`, `river`, `ruins`) - служебное имя **для системы и ведущего**. Игроки его **не видят**.

Зачем нужен:

- у каждой сцены в игре уникальный внутренний id;
- ведущий в консоли может ориентироваться: «сцена `rapids`»;
- позже: ссылки, картинки, лог событий.

Правила: латиница, цифры, `_` и `-`, без пробелов.  
Примеры: `briefing`, `camp_night`, `act2-ruins`.

**Аналогия:** номер и название главы в книге. Читатель видит текст главы, а «Глава 3» - для автора и оглавления.

### Тип сцены - зачем три варианта

Сейчас в редакторе три типа. Это **подсказки для ведущего**, не жёсткие правила движка.

| Тип                  | Простыми словами                                                                        | Игроки видят                                                                  |
| -------------------- | --------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------- |
| **Сюжет (STORY)**    | Обычный момент истории: описание места, событие, диалог                                 | Фон при старте сцены; текст и задания - когда ведущий **откроет** их отдельно |
| **Проверка (CHECK)** | Момент, где может понадобиться кубик: «кто заметит след?», «удастся ли починить лодку?» | То же, что сюжет + ведущий может запросить бросок                             |
| **Заметка (NOTE)**   | Шпаргалка только для ведущего: список концовок, имена NPC, подсказки по сложности       | **Ничего** - NOTE не запускается как сцена для игроков                        |

**Важно:** тип не блокирует игру. NOTE не отправляется игрокам автоматически. CHECK не бросает кубик сам - ведущий решает.

### Поля сцены в редакторе

| Поле                           | Кто видит                                                    | Зачем                                                 |
| ------------------------------ | ------------------------------------------------------------ | ----------------------------------------------------- |
| **Текст сцены** (RU/EN)        | Игроки, когда ведущий нажал **«Показать текст»**             | Читать с экрана вслух или про себя                    |
| **Фон сцены** (загрузка файла) | Игроки сразу при **«Начать сцену»**                          | Атмосфера: джунгли, лагерь, река                      |
| **Картинки сцены** (несколько) | Когда ведущий нажмёт **«Показать картинку»**                 | Волк, преграда, находка - фон при этом остаётся       |
| **Задания** (можно несколько)  | Игроки, когда ведущий нажал **«Показать задание»** у каждого | Короткие задачи для всех; каждое открывается отдельно |
| **Заметки ведущего**           | Только ведущий                                               | Спойлеры, цифры, «если провал - вот что случится»     |

Текст и задания **не показываются автоматически** при старте сцены. Ведущий сам решает, что и когда открыть.

### Сцены через AI (редактор)

Ведущий **не обязан** писать все сцены вручную:

1. Заполняет название, героев, характеристики → **Сохранить**
2. В блоке «Сцены через AI» → **Сгенерировать промпт** (учитывает всё заданное + ваши правки в `[скобках]`)
3. Копирует промпт в любой чат (ChatGPT, Claude…)
4. Вставляет JSON-ответ обратно → **Разобрать черновик** → правит → **Сохранить сцены в игру**

Формат ответа AI описан внутри промпта. API к AI в приложении **нет** - только промпт и импорт.

### Как ведущий ведёт одну сцену (пошагово)

1. **Начать сцену** - у игроков: «Сцена N началась» + **фон** (если загружен). Текста, картинок и заданий пока нет.
2. **Показать текст** / **Скрыть текст** - текст сцены появляется или исчезает у всех.
3. **Показать картинку** / **Скрыть картинку** - у каждой картинки своя кнопка; фон остаётся.
4. **Показать задание** / **Скрыть задание** - у каждого задания своя кнопка.
5. **Завершить сцену** - сцена уходит в «Завершена»; можно начать следующую.

В консоли ведущего есть **таблица прогресса**: все сцены шаблона со статусами «Ожидает», «Идёт», «Завершена».

В редакторе сцены: **фон** и отдельный список **картинок**. Локально - Docker (`docker compose up`); на сервере - те же переменные `S3_*`, но со своими значениями (R2, S3).

## Как выглядит партия целиком

```
Игрок: код комнаты + имя
    ↓
Лобби (ждёт старта)
    ↓
Выбор героя из списка, который задал ведущий в шаблоне
    ↓
Карточка персонажа: роль + генерация характеристик
    ↓
«Готов» → ждёт, пока ведущий начнёт игру
    ↓
Ведущий: начать сцену → (опционально) показать текст → показать задания по одному
    ↓
Следующие сцены по тому же принципу; таблица прогресса у ведущего
    ↓
Иногда: проверка кубиком
    ↓
Финал (ведущий ведёт устно)
```

### Что уже работает

- Вход по коду + **уникальное имя** в комнате
- Лобби, старт сессии ведущим (минимум 1 игрок)
- Редактор шаблона: название, сцены (текст, фон URL, задания, заметки), слоты героев, характеристики
- Лобби игрока: выбор героя → карточка → случайные статы → «Готов»
- Ведущий видит карточки игроков в лобби и во время игры
- Попап «Моя карточка» во время игры
- Пошаговый показ сцен: старт, текст, задания, прогресс по сценам
- Импорт сцен через AI (промпт + JSON)

### Что делаем дальше

1. Загрузка картинок в редактор (сейчас только URL)
2. Блокировка старта, пока не все игроки «Готов» (опционально)
3. Личные задания отдельным игрокам из консоли

## Персонажи и герои (ваше видение)

### Кто настраивает

**Ведущий** в шаблоне игры, до сессии собирает **список слотов** (не «типы», а конкретные места за столом):

1. Включает готовые архетипы галочками: врач, инженер, журналист…
2. Может **добавить своих** (поле + «добавить»)
3. Может **дублировать** одну и ту же роль несколько раз, если игроков больше, чем уникальных профессий

**Пример:** в справочнике 6 архетипов, а играют 8 человек → ведущий добавляет в список два раза «Врач», один раз «Инженер» и т.д. В итоге в шаблоне **8 слотов**, среди них два врача - но это **два разных слота**.

| Понятие            | Значение                                                                                   |
| ------------------ | ------------------------------------------------------------------------------------------ |
| **Слот**           | Одна строка в списке ведущего (можно назвать «Врач», «Врач (2)» или просто два «Врач»)     |
| **Сколько слотов** | Столько игроков с персонажами (8 слотов → до 8 человек с карточками)                       |
| **Выбор игрока**   | Берёт **один свободный слот** из списка. Занятый слот другим недоступен                    |
| **Два врача**      | Два слота с одинаковым названием - ок. Первый врач занят → второй слот «врач» ещё свободен |

Список **не общий на всю платформу** - только в этом шаблоне. Другая игра = другой набор слотов.

### Сильная сторона и слабость (у каждого слота)

Помимо случайных характеристик у **каждого слота героя** ведущий задаёт:

| Поле                | Смысл                                            | Пример                |
| ------------------- | ------------------------------------------------ | --------------------- |
| **Сильная сторона** | То, в чём этот герой стабильно силён (~35 очков) | Врач → Медицина       |
| **Слабость**        | То, где герой уязвим (~8 очков)                  | Врач → Импульсивность |

Правила:

- у **каждого слота** ровно одна сильная сторона и одна слабость;
- названия **не повторяются** между слотами в одной игре (два врача - две разные пары сила/слабость);
- сильная сторона и слабость **одного** героя не совпадают;
- при дублировании слота ведущий меняет пару, иначе сохранение не пройдёт.

Остальные характеристики из общего списка игрок получает **случайно** при входе в сессию.

### Что делает игрок при входе

1. Ввёл код и имя (как сейчас)
2. Видит: **«Выберите героя»** - свободные **слоты** из списка ведущего
3. **Один слот = один игрок.** Кто-то занял слот «Врач #1» - этот слот недоступен; слот «Врач #2» (если ведущий добавил второго) - ещё можно взять
4. Сколько **слотов** в шаблоне - столько персонажей в партии (8 слотов → 8 игроков могут выбрать себе героя). Каждый слот в списке выбирается **только один раз**
5. После выбора открывается **карточка**:
   - название роли;
   - **фиксированная** сильная сторона и слабость слота;
   - аватар (позже, не в первой версии);
   - общие характеристики (после случайной генерации)
6. **«Перегенерировать»** статы - сколько угодно, пока не нажал **«Готов»**. После «Готов» - цифры зафиксированы
7. В игре в любой момент - кнопка **«Моя карточка»** (модальное окно)

Игрок **не придумывает** роль с нуля - выбирает из списка ведущего. Так проще и честнее для сюжета.

### Характеристики (общий пул)

**Ведущий** в шаблоне:

- выбирает из **готового списка** (Наблюдательность, Выносливость, Убеждение, Выживание, Медицина…);
- минимум **3** характеристики на игру;
- может добавить свои названия;
- всё это только для **этой** игры.

Это **не** сильная сторона и слабость слота - они задаются отдельно у каждого героя.

Игрок **не вводит** цифры руками. Один раз нажимает **«Сгенерировать»** - приложение раскидывает очки по общему списку. Сильная сторона и слабость уже на карточке и не меняются.

## Генерация очков

Цель: у каждого игрока разный «билд», но без экстремальных перекосов (70 в одной характеристике при максимуме 30).

### Правила

| Параметр                       | Значение                                                            |
| ------------------------------ | ------------------------------------------------------------------- |
| **Максимум на характеристику** | Задаёт ведущий в шаблоне (по умолчанию 30)                          |
| **При генерации**              | Каждая общая характеристика получает **случайно от 0 до максимума** |
| **Суммарный пул**              | Нет - очки не делятся между характеристиками                        |
| **Повтор**                     | Кнопка «перегенерировать» до **«Готов»**; после - фикс              |

Сильная сторона и слабость героя задаются отдельно у слота и не участвуют в этой генерации.

### Связь с проверками в игре

Когда ведущий объявляет проверку («бросьте Наблюдательность»), к броску кубика добавляется значение этой характеристики. Это проще, чем полные правила D&D.

## Что видит ведущий во время сессии

- **Таблица прогресса сцен** - все сцены шаблона: Ожидает / Идёт / Завершена
- Список игроков: имя, герой, карточка с характеристиками
- В лобби: кто готов, кто выбирает героя / крутит статы
- **Текущая сцена**: заметки ведущего, статус (текст виден или скрыт, сколько заданий на экране)
- Кнопки у активной сцены: **Начать сцену**, **Показать/скрыть текст**, **Показать/скрыть задание** (по каждому), **Завершить сцену**
- Запрос броска кубика
- Лог событий (в движке)

### Задания игрокам

| Уровень             | Как                                                                    |
| ------------------- | ---------------------------------------------------------------------- |
| **В шаблоне сцены** | Несколько заданий на сцену; ведущий открывает каждое отдельной кнопкой |
| **Устно**           | Как и раньше - ведущий может говорить без полей в редакторе            |
| **Позже**           | Личное задание каждому игроку из консоли                               |

## Чеклист: грамотный шаблон перед первой партией

### 1. Описание игры

- [ ] Понятно за 2 предложения: где, кто, чего хотят
- [ ] Длина: ориентир 60-90 минут (можно написать в описании)

### 2. Герои

- [ ] Столько **слотов**, сколько планируете игроков (можно два одинаковых названия)
- [ ] У каждого слота своя пара сила/слабость (не повторяются в игре)
- [ ] Роли понятны в сюжете; дубликаты - осознанно («два медика в экспедиции»)

### 3. Характеристики

- [ ] Минимум 3, названия понятны без правил
- [ ] Хотя бы одна пригодится в каждой трети сцен

### 4. Сцены

- [ ] Первая сцена - вступление (кто вы, зачем собрались)
- [ ] Каждая сцена - **одна** ясная ситуация
- [ ] Есть напряжение в середине и развязка в конце
- [ ] Заметки ведущего: имена, цифры, что если игроки застряли
- [ ] Задания короткие; если их несколько - каждое про отдельный шаг сюжета

### 5. Картинки

- [ ] Хотя бы ключевые сцены с URL фона (загрузка файлов - позже)

### 6. Перед публикацией

- [ ] Минимум 1 сцена (технически)
- [ ] Прогон мысленно: игрок зашёл → выбрал героя → сгенерировал статы → 3 сцены подряд понятны

## Сравнение с карточкой D&D (кратко)

| D&D                                     | Game Harbour                    |
| --------------------------------------- | ------------------------------- |
| Раса, класс, уровень, заклинания        | Роль из списка ведущего         |
| 6 характеристик, модификаторы, владения | 3+ простых параметра, пул очков |
| Книга правил                            | Ведущий + короткий шаблон       |
| Лист персонажа на бумаге                | Карточка в телефоне             |

Мы **не заменяем** D&D для тех, кто любит D&D. Мы делаем **лёгкий сюжетный формат** с телефоном у игрока и консолью у ведущего.

## Решено (подтверждено)

| #   | Решение                                                                                                                                                             |
| --- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 1   | **Список слотов от ведущего.** Можно дублировать роли (2 врача). Каждый **слот** выбирается только один раз; занятый недоступен. N слотов ≈ N игроков с персонажами |
| 2   | **Перегенерировать статы** можно до кнопки **«Готов»**, после - нельзя                                                                                              |
| 3   | **Ведущий видит карточки** игроков (герой + характеристики) в лобби и во время игры                                                                                 |
| 4   | **Сцена:** **«Начать сцену»** не показывает текст; фон - сразу; текст и задания - отдельными кнопками                                                               |
| 5   | **Задания в сцене:** несколько на сцену; каждое показывается отдельно; личные задания - позже                                                                       |
| 6   | **Прогресс сцен:** таблица у ведущего (ожидает / идёт / завершена)                                                                                                  |
